# Arsitektur Decision Record

## Kapan membuat ADR
Buat ADR bila keputusan:
- memengaruhi lebih dari satu modul atau layer;
- mengubah pola arsitektur yang sebelumnya umum;
- memperkenalkan read model, materialization, atau strategi akses data baru;
- mengubah policy retry, rerun, atau idempotensi penting;
- menambah variasi resmi pada struktur folder atau boundary;
- memperkenalkan pengecualian yang cukup besar.
