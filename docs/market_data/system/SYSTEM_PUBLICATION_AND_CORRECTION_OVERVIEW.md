# System Publication and Correction Overview

## Publication principle
Data hanya dianggap published bila quality gates, manifest requirements, dan publication procedures yang relevan terpenuhi.

## Correction principle
Historical correction bukan overwrite diam-diam. Correction harus mengikuti contract correction/reseal/replay dan tetap menjaga auditability.

## Consumer impact principle
Consumer downstream harus bisa membedakan dengan jelas:
- published current state
- corrected state
- historical state
- readiness / partial-data caveats

## Detail authority
Detail final tetap ada di contract docs dan ops docs terkait publication, correction, reseal, dan pointer switching.
